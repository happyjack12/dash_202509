# Аналитический дашборд магазина велосипедов

## Описание проекта
Данный проект представляет собой интерактивный аналитический инструмент для мониторинга и визуализации ключевых показателей продаж магазина велосипедов. Проект выполнен в рамках учебного курса по аналитике данных и разработке дашбордов.

**Автор:** [Халимов Саъди]

---

## Описание датасета
В основе проекта лежит набор данных **Bike Stores Sample Database**, представляющий собой структуру транзакционных данных розничной сети.

* **Источник:** [[Ссылка]](https://www.kaggle.com/datasets/dillonmyrick/bike-store-sample-database)
* **Состав данных:** Датасет содержит информацию о клиентах, магазинах, категориях товаров, брендах, характеристиках велосипедов и детальную историю заказов.
* **Цель анализа:** Выявление наиболее прибыльных категорий и брендов, отслеживание накопительной выручки и анализ взаимосвязи между объемом продаж и доходом.

## Структура репозитория
Репозиторий организован по модульному принципу для разделения логики обработки данных, SQL-запросов и визуализации:

* `main.py` — основной файл приложения Streamlit, отвечающий за интерфейс и логику фильтрации.
* `db.py` — модуль для подключения к базе данных DuckDB и выполнения запросов.
* `ddl.py` — ETL-скрипт (Pipeline) для автоматического создания таблиц и загрузки данных из CSV-файлов.
* `my.db` — локальная база данных формата DuckDB.
* `queries/` — папка с SQL-файлами:
    * `*.sql` — DDL-запросы на создание таблиц.
    * `sales_analytics.sql` — запрос для основной витрины данных (Join, Агрегации).
    * `customer_dynamics.sql` — запрос с использованием оконных функций.
    * `columns.json` — конфигурация столбцов для сопоставления данных.
* `source/` — исходные данные в формате CSV.
* `requirements.txt` — список необходимых Python-библиотек.

## Используемые инструменты
Для реализации проекта был использован современный стек технологий обработки данных:

* **Python 3.10+**: Основной язык разработки.
* **Streamlit**: Фреймворк для создания веб-интерфейса дашборда.
* **DuckDB**: Быстрая аналитическая (OLAP) база данных внутри приложения.
* **Pandas**: Обработка, очистка и подготовка данных (ETL).
* **Plotly Express**: Создание интерактивных графиков.
* **SQL**: Написание аналитических запросов.

### Как запустить локально:
1. Клонируйте репозиторий.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Запустите загрузку данных в базу: `python ddl.py`.
4. Запустите дашборд: `streamlit run main.py`.
